# Руководство пользователя ИАС «Железнодорожный перевозчик»

## Как пользоваться системой (общая инструкция)


### 2. СТРУКТУРА ИНТЕРФЕЙСА

Приложение имеет три вкладки (вверху страницы):
- **🚂 Оценка технического уровня** — первая вкладка
- **📈 Прогноз пассажиропотока** — вторая вкладка
- **🛤️ Оптимизация перевозок** — третья вкладка

Каждая вкладка состоит из двух колонок:
- **Левая колонка** — для ввода данных и настроек
- **Правая колонка** — для отображения результатов

## ОЦЕНКА ТЕХНИЧЕСКОГО УРОВНЯ

### Как оценить качество подвижного состава

1. **Выберите тип подвижного состава**
   - Найдите выпадающий список «Выбор типа подвижного состава» в левой колонке
   - Выберите один из трёх типов:
     - 🚂 Пассажирские электровозы (ЭП20, 2ЭС6, ВЛ85)
     - 🛤️ Грузовые тепловозы (2ТЭ25К, ТЭМ18, 2М62)
     - 🚄 Электропоезда (пригородные) (ЭД4М, ЭГ2Тв, Ласточка)

2. **Добавьте образцы для сравнения** (выберите один из способов):

   **Способ А: Ручное добавление**
   - Найдите блок «Ручное добавление образца»
   - Введите название модели (например, «ЭП20»)
   - Заполните все показатели (скорость, ресурс, мощность, надёжность, комфорт, расход топлива)
   - Нажмите кнопку «➕ Добавить образец»
   - Повторите для каждого образца

   **Способ Б: Загрузка из CSV-файла**
   - Подготовьте CSV-файл с колонками: name, max_speed, resource, traction_power, reliability, comfort, fuel_consumption
   - Перетащите файл в область «Загрузка данных» или нажмите для выбора
   - Файл загрузится автоматически

3. **Запустите расчёт**
   - Нажмите кнопку «📊 Рассчитать технический уровень»
   - Дождитесь завершения расчёта

4. **Посмотрите результаты** (в правой колонке):
   - **Радиальная диаграмма** — сравнивает все образцы по каждому параметру
     - Значение 1.0 = эталон
     - больше 1.0 = лучше эталона
     - меньше 1.0 = хуже эталона
   - **Столбчатая диаграмма** — показывает рейтинг образцов в процентах
   - **Рейтинг образцов** — текстовый список от лучшего к худшему

5. **Интерпретация результатов**:
   - Чем выше столбец на диаграмме, тем лучше локомотив
   - Чем больше площадь фигуры на радиальной диаграмме, тем лучше локомотив
   - Образец с индексом > 100% превосходит эталон

## ПРОГНОЗ ПАССАЖИРОПОТОКА

### Как получить прогноз пассажиров

1. **Откройте вкладку «Прогноз пассажиропотока»**

2. **Введите данные для прогноза** в левой колонке:
   - 🎫 **Цена билета** — укажите стоимость билета (руб.)
   - 🛤️ **Протяжённость маршрута** — укажите длину маршрута (км)
   - 📅 **Выходной/праздник** — введите 1 (да) или 0 (нет)
   - 🚆 **Количество поездов** — укажите количество поездов в сутки
   - 🌤️ **Сезон** — введите число: 1-зима, 2-весна, 3-лето, 4-осень

3. **Получите прогноз**:
   - Нажмите кнопку «📈 Получить прогноз»
   - Результат появится под кнопкой

4. **Оцените качество модели** (информация над полями ввода):
   - **R²** — чем ближе к 1, тем точнее прогноз (хорошо > 0.95)
   - **MAE** — средняя ошибка в тыс. человек (чем меньше, тем лучше)

### Как построить график зависимости

1. В правой колонке найдите блок «Анализ зависимости пассажиропотока»

2. **Выберите признак**:
   - Цена билета
   - Протяжённость маршрута
   - Количество поездов

3. **Задайте диапазон** значений для анализа

4. Нажмите «📊 Построить график зависимости»

5. **Прочитайте график**:
   - По оси X — выбранный признак
   - По оси Y — прогнозируемый пассажиропоток
   - Остальные признаки зафиксированы на средних значениях

## ОПТИМИЗАЦИЯ ПЕРЕВОЗОК

### Как оптимизировать распределение локомотивов

1. **Откройте вкладку «Оптимизация перевозок»**

2. **Проверьте исходные данные**:
   - **Матрица затрат** — таблица 3×5 (3 депо × 5 направлений)
     - Строки — депо (ТЧ-1, ТЧ-2, ТЧ-3)
     - Столбцы — направления (Москва-СПб, Москва-Казань и т.д.)
     - Значения — стоимость в руб./км
   - **Запасы локомотивов** — сколько локомотивов в каждом депо
   - **Потребности направлений** — сколько локомотивов нужно на каждом направлении

3. **При необходимости отредактируйте данные**:
   - Кликните на любую ячейку матрицы затрат и измените число
   - Измените значения в полях «Запасы локомотивов в депо»
   - Измените значения в полях «Потребность направлений»

4. **Запустите оптимизацию**:
   - Нажмите кнопку «🚀 Оптимизировать распределение»
   - Дождитесь завершения расчёта

5. **Посмотрите результаты** (в правой колонке):
   - **Минимальные затраты** — общая стоимость в руб./км
   - **Таблица распределения** — сколько локомотивов из каждого депо на каждое направление
   - **Сравнение методов** — время выполнения трёх методов оптимизации
   - **Тепловая карта** — визуализация матрицы затрат (зелёный = дёшево, красный = дорого)

6. **Интерпретация результатов**:
   - Чем меньше затраты, тем лучше план
   - Таблица показывает, сколько локомотивов куда отправить
   - Зелёные ячейки на тепловой карте — выгодные маршруты

### Что делать, если задача не сбалансирована

Если сумма запасов ≠ сумме потребностей:
- Программа **автоматически** добавит фиктивное направление или фиктивное депо
- Вы увидите сообщение об этом в результатах
- Вам не нужно ничего делать — всё настроится автоматически

## ОБЩИЕ СОВЕТЫ ПО РАБОТЕ

### 1. Что делать, если ничего не происходит

1. Проверьте, что вы нажали нужную кнопку:
   - «Рассчитать технический уровень» — для оценки качества
   - «Получить прогноз» — для прогнозирования
   - «Оптимизировать распределение» — для оптимизации

2. Проверьте, что все поля заполнены (для прогноза — все 5 полей)

3. Перезагрузите страницу (F5)

4. Перезапустите приложение:
   - Закройте терминал (Ctrl+C)
   - Снова запустите: `python main.py`

### 2. Как интерпретировать цвета на графиках

**Радиальная диаграмма:**
- Синий, красный, зелёный и другие цвета — разные образцы
- Каждый цвет = отдельный локомотив или поезд

**Столбчатая диаграмма:**
- Зелёный столбец — лучший образец
- Синие столбцы — остальные образцы

**Тепловая карта:**
- Зелёный — низкие затраты (выгодно)
- Жёлтый — средние затраты
- Красный — высокие затраты (невыгодно)

### 3. Какие форматы данных поддерживаются

**Для оценки качества (CSV):**

Для прогноза (используется автоматически):

Ответ: Вкладка «Оценка технического уровня» → блок «Загрузка данных» → перетащить CSV-файл → «Рассчитать технический уровень»
