# Полная документация по ИАС Железнодорожный перевозчик

## 1. ОБЩЕЕ ОПИСАНИЕ СИСТЕМЫ

**Название:** Информационно-аналитическая система (ИАС) «Железнодорожный перевозчик»

**Назначение:** Автоматизация анализа и оптимизации работы железнодорожного перевозчика. Система помогает:
- Оценивать технический уровень подвижного состава
- Прогнозировать пассажиропоток
- Оптимизировать распределение локомотивов по направлениям

**Технологии:** Python 3.10+, Dash (веб-интерфейс), Plotly (графики), scikit-learn (ML), SciPy (оптимизация)

---

## 2. МОДУЛЬ 1: ОЦЕНКА ТЕХНИЧЕСКОГО УРОВНЯ

### 2.1. Назначение
Позволяет сравнивать различные модели подвижного состава по техническим характеристикам и определять лучший образец.

### 2.2. Типы подвижного состава
| Тип | Примеры |
|-----|---------|
| Пассажирские электровозы | ЭП20, 2ЭС6, ВЛ85 |
| Грузовые тепловозы | 2ТЭ25К, ТЭМ18, 2М62 |
| Электропоезда (пригородные) | ЭД4М, ЭГ2Тв, Ласточка |

### 2.3. Характеристики для оценки
| Параметр | Описание | Тип |
|----------|----------|-----|
| max_speed | Максимальная скорость (км/ч) | Стимулятор ↑ |
| resource | Ресурс до капремонта (тыс. км) | Стимулятор ↑ |
| traction_power | Мощность (кВт) | Стимулятор ↑ |
| reliability | Надёжность (баллы 0-10) | Стимулятор ↑ |
| comfort | Комфорт (баллы 0-10) | Стимулятор ↑ |
| fuel_consumption | Расход топлива (л/100км) | Дестимулятор ↓ |

### 2.4. Как это работает
1. Пользователь выбирает тип подвижного состава
2. Добавляет образцы (вручную или через CSV)
3. Система сравнивает каждый образец с эталоном
4. Рассчитывается интегральный индекс качества
5. Результаты отображаются на графиках

### 2.5. Способы добавления образцов

**Способ А: Ручной ввод**
1. Заполните поля в блоке «Ручное добавление образца»
2. Нажмите «Добавить образец»
3. Повторите для всех образцов
4. Нажмите «Рассчитать технический уровень»

**Способ Б: Загрузка CSV**
1. Подготовьте файл с колонками: name, max_speed, resource, traction_power, reliability, comfort, fuel_consumption
2. Перетащите файл в область загрузки
3. Нажмите «Рассчитать технический уровень»

### 2.6. Интерпретация результатов

**Радиальная диаграмма:**
- Каждая ось = один параметр
- 1.0 = эталон
- > 1.0 = лучше эталона
- < 1.0 = хуже эталона
- Чем больше площадь фигуры, тем лучше образец

**Столбчатая диаграмма:**
- Показывает интегральный индекс качества в %
- Чем выше столбец, тем лучше образец
- Первое место — лучший образец

**Эталон для каждого типа:**
- Пассажирские электровозы: ЭП20
- Грузовые тепловозы: 2ТЭ25К
- Электропоезда: Ласточка

## 3. МОДУЛЬ 2: ПРОГНОЗ ПАССАЖИРОПОТОКА

### 3.1. Назначение
Прогнозирование количества пассажиров на основе различных факторов.

### 3.2. Входные параметры
| Параметр | Описание | Диапазон |
|----------|----------|----------|
| ticket_price | Цена билета (руб.) | 700-1050 |
| route_length | Протяжённость маршрута (км) | 80-400 |
| is_weekend | Выходной/праздник | 0 или 1 |
| trains_per_day | Количество поездов в сутки | 6-26 |
| season | Сезон | 1-зима, 2-весна, 3-лето, 4-осень |

### 3.3. Как это работает
1. Данные загружаются из Passenger.csv
2. Модель (полиномиальная регрессия) обучается на 80% данных
3. Качество проверяется на 20% тестовых данных
4. Пользователь вводит значения → получает прогноз

### 3.4. Метрики качества
| Метрика | Что показывает | Хорошее значение |
|---------|----------------|------------------|
| R² | Доля объяснённой дисперсии | > 0.95 |
| MAE | Средняя ошибка (тыс. чел.) | < 15 |

### 3.5. График зависимости
Позволяет увидеть, как пассажиропоток зависит от выбранного признака:
1. Выберите признак (цена, длина, количество поездов)
2. Задайте диапазон значений
3. Нажмите «Построить график»

## 4. МОДУЛЬ 3: ОПТИМИЗАЦИЯ ПЕРЕВОЗОК

### 4.1. Назначение
Оптимальное распределение локомотивов из депо по направлениям с минимальными затратами.

### 4.2. Исходные данные

**Депо (поставщики):**
| Депо | Локомотивов |
|------|-------------|
| ТЧ-1 | 30 |
| ТЧ-2 | 25 |
| ТЧ-3 | 20 |

**Направления (потребители):**
| Направление | Потребность |
|-------------|-------------|
| Москва-СПб | 15 |
| Москва-Казань | 12 |
| Москва-Екатеринбург | 18 |
| Москва-Новосибирск | 14 |
| Москва-Владивосток | 16 |

**Матрица затрат (руб./км):**
| Депо \ Направление | М-СПб | М-Кзн | М-Екб | М-Нсб | М-Влк |
|-------------------|-------|-------|-------|-------|-------|
| ТЧ-1 | 500 | 550 | 600 | 650 | 700 |
| ТЧ-2 | 580 | 520 | 570 | 620 | 670 |
| ТЧ-3 | 650 | 600 | 550 | 600 | 650 |

### 4.3. Как это работает
1. Задача считается сбалансированной (75 = 75)
2. Симплекс-метод находит оптимальное распределение
3. Сравниваются три метода оптимизации
4. Выводится результат с минимальными затратами

### 4.4. Что можно редактировать
- Матрицу затрат (любую ячейку)
- Запасы локомотивов в депо
- Потребности направлений

### 4.5. Результаты
- Таблица распределения (сколько локомотивов куда)
- Минимальные затраты (руб./км)
- Сравнение методов оптимизации
- Тепловая карта матрицы затрат

### 4.6. Сбалансированность
Задача сбалансирована, когда сумма запасов = сумме потребностей.
В системе: 30+25+20 = 15+12+18+14+16 = 75

Если баланс нарушен, система автоматически добавляет фиктивного поставщика или потребителя с нулевыми затратами.
